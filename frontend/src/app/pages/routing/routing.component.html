Documentation sur les routers en Angular

Qu'est-ce qu'un router en Angular ?

Le Router en Angular est un outil qui gère la navigation et le chargement des composants en fonction de l'URL de la page. Contrairement aux applications web traditionnelles, où chaque navigation recharge une page entière, Angular utilise un router pour permettre de changer de contenu dynamiquement sans recharger toute la page. C'est ce qu'on appelle une single page application.

Configuration d'un router

Dans Angular, les routes sont définies sous forme de tableau d'objets, où chaque objet correspond à une route spécifique. Ceci ce fait dans le fichier routes.ts: 

{{ configRouterExemple }}

Chaque objet de route contient les propriétés suivantes :

  • Path : le chemin de l'URL qui déclenchera la route
  • Component : le composant qui sera chargé lorsque le chemin de l'URL correspond à la route

Utilisation d'un router

Pour pouvoir utiliser toutes ces rajoutes rajoutées dans notre liste, dans le fichier main.ts généré automatiquement lors de l'initialisation du projet, il faut rajouter la ligne suivante : 

{{ routerMainExemple }}

Angular utilise un composant spécial appelé <router-outlet> pour afficher dynamiquement le contenu associé à une route. 

{{ routerLinkExemple }}

La raison pour laquelle on utilise le routerLink est pour pouvoir naviguer entre les différentes pages de notre application. Il est préféré à href car il permet une navigation sans recharger la page entière, ce qui est crucial pour les applications SPA.

Lorsque vous utilisez un lien avec href, le navigateur effectue un rechargement complet de la page.

Gestion des routes paramétrées

Il est souvent utile d'inclure des paramètres dans les routes pour permettre une navigation dynamique, par exemple, afficher les informations d'un utilisateur spécifique ou d'un produit particulier. Cela se fait en ajoutant des variables dynamiques dans le chemin des routes, définies avec le préfixe :.

Voici un exemple de définition de routes paramétrées dans le fichier routes.ts :

 {{ rParamExemple }}

Dans l'exemple ci-dessus, la route /user/:id accepte un paramètre dynamique nommé id. Ce paramètre peut être utilisé dans le composant correspondant pour charger des données spécifiques à cet utilisateur.

Récupération des paramètres dans le composant

Dans le composant correspondant, vous pouvez récupérer les paramètres dynamiques via le service ActivatedRoute fourni par Angular. 

 {{ recupParamExemple }}

Dans cet exemple, la méthode snapshot.paramMap.get('id') permet de récupérer la valeur du paramètre id présent dans l'URL au moment du chargement du composant. Vous pouvez ensuite utiliser cette valeur pour afficher ou charger des données spécifiques, par exemple depuis une API ou une base de données.

Navigation vers une route paramétrée

Pour naviguer vers une route paramétrée, utilisez la directive routerLink avec des paramètres dynamiques : 

{{ navVersRouteExemple }}

Ce lien redirige l'utilisateur vers la route /user/user.id et charge le composant associé avec l'ID passé comme paramètre.
